# Aegis Ops: Intelligent Engineering Operations Platform

An enterprise-ready engineering operations platform for microservice monitoring, raw log indexing, automated anomaly detection, incident prioritization, and pre-deployment build risk analytics.

Aegis Ops integrates a statistical and machine learning pipeline that filters out alert noise, identifies anomalies before they cause widespread outages, and streamlines incident remediation for engineering teams.

---

## 🌟 Core Modules & Features

### 1. Service Monitoring Dashboard
- Displays real-time health grids tracking CPU, memory allocation, request counts, response latency, and error rates.
- Displays container status indicators (`healthy`, `warning`, `critical`) driven by live metrics.

### 2. Anomaly Detection (ML Engine)
- Employs **rolling Z-Scores** and **Interquartile Range (IQR)** to detect outliers in performance metrics.
- Leverages an **Isolation Forest** multivariate model (via `scikit-learn`) to detect complex multi-metric anomalies.
- Explains anomalies with detailed summaries (e.g., *“Response Latency is 3.4x higher than the 60min moving average”*).

### 3. Alert Prioritization System
- Evaluates and ranks anomalies using a dynamic scoring algorithm:
  $$\text{Priority Score} = \text{Metric Outlier Magnitude} \times \text{Service Tier Factor} + \text{Error Rate Penalty}$$
- Tiers alerts into `Low`, `Medium`, `High`, and `Critical` priorities, displaying the highest severity issues first.

### 4. Log Intelligence Terminal
- Monospace CLI log console streaming service events.
- Advanced filters for Service, Severity, and text search.
- **ML Scanning mode** highlighting security threats (such as brute-force attacks or SQL injection) and performance delays (such as database deadlocks or heap limits) using NLP classification.

### 5. Incident Kanban Board & Notes
- Outage tickets are automatically generated by the ML engine for High/Critical anomalies.
- Drag-and-drop styled Kanban board tracking status through `Open`, `Investigating`, and `Resolved`.
- Incident rooms include timeline notes, action logs, and comments for collaborative debug operations.

### 6. Pre-deployment Risk Calculator
- Analyzes potential deployment risk before code rollouts by combining target service tier weight, recent metric instability, previous rollout failure rates, and natural language analysis of changes in git branch comments.
- Provides dynamic gauges and lists specific identified risk factors.

### 7. Access Control (Console Authority)
- Simulates RBAC permissions:
  - **Viewer**: Read-only dashboard access.
  - **Operator**: Authorized to acknowledge alerts, write debug notes, and resolve incidents.
  - **Admin**: Authorized to adjust baseline thresholds, execute code deployments, and reset platform databases.

## 🛠 Tech Stack

- **Frontend**: React 19, Vite, SVG-based graphing, Custom CSS (Glassmorphism & Neon elements).
- **Backend**: FastAPI (Python), Uvicorn.
- **Database**: SQLite (SQL query history, log indices, metrics series).
- **ML Libraries**: NumPy, Re (regex NLP parser), Scikit-Learn (Isolation Forest).

## 🚀 Setup & Execution

### Prerequisites
Ensure you have **Python 3.8+** and **Node.js 16+** installed.

### 1. Backend Server Setup
1. Navigate to the backend directory:
   ```bash
   cd backend
   ```
2. Install Python dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Initialize and seed the SQLite database:
   ```bash
   python database.py
   ```
4. Start the FastAPI Uvicorn server:
   ```bash
   python -m uvicorn main:app --port 8000 --reload
   ```
   *The backend will run on [http://localhost:8000](http://localhost:8000).*

### 2. Frontend Console Setup
1. Navigate to the frontend directory:
   ```bash
   cd ../frontend
   ```
2. Install Node dependencies:
   ```bash
   npm install
   ```
3. Start the Vite development server:
   ```bash
   npm run dev
   ```
   *The UI will be accessible on [http://localhost:5173](http://localhost:5173).*

## 📈 ML Threshold Feedback Loop

Aegis Ops includes a **reinforcement feedback loop**:
1. When the ML scanner triggers an anomaly, operators can click **✓ Correct Anomaly** or **✗ False Positive**.
2. Submitting feedback tells the backend to adjust the metric sensitivity multiplier.
3. If marked as a *False Positive*, the model dynamically raises the Z-score anomaly limit by 25% for that service, preventing repeated alert noise.
